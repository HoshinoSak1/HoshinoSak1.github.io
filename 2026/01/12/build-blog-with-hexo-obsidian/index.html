GitPage

Hexo 完成了离线内容的搭建，那么具体的博客内容需要存在线上托管，正好 GitHub 提供 GitPage 来实现快速搭建博客的功能，做法如下：

  1. 在 GitHub 创建仓库
    1. 登录你的 GitHub 账号。
    2. 点击右上角的 + 号，选择 New repository。
    3. Repository name（关键）：
      • 必须填为：你的用户名.github.io
      • 例如：如果你的 GitHub 用户名是 zhangsan，仓库名必须是 zhangsan.github.io。
      • 解释：这是 GitHub Pages 的特殊规则，只有这样命名，你的博客地址才会是简洁的 https://zhangsan.github.io。
    4. 确保选择 Public（公开）。
    5. 需要注意的是，这里 GitPage 的官方教程会教你打开 Add README 为 On，但是基于 Hexo 进行搭建时，这里需要确保选择的是 Off，后续可以规避一部分配置。
    6. 点击 Create repository。
  2. 配置网页仓来源，这里遵循 GitPage 教程即可。
    1. 在你的仓库名称下，点击设置 。如果看不到“设置”标签，选择下拉菜单，然后点击设置 。
    2. 在侧边栏的“代码与自动化”部分，点击页面 。
    3. 在“构建和部署”中，在“源”下，从分支选择部署 。
    4. 在“构建和部署”中，在“分支”下，使用分支下拉菜单选择发布源。需要注意这里的发布源是main还是master ，后续配置有用。 Img page
